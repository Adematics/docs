    Delete 

    type: "io.kestra.plugin.fs.ftps.Delete"

    Delete a file to a FTPS server.

    # Examples

    id: "delete"
type: "io.kestra.plugin.fs.ftps.Delete"
host: localhost
port: 990
username: foo
password: pass
uri: "/upload/dir1/file.txt"

    # Properties

    # dataChannelProtectionLevel

    • Type: string

    • Dynamic: ❓

    • Required: ❌

    • Default: P

    • Possible Values:

      • C
      • S
      • E
      • P

    Sets the data channel protection level (PROT).

    # errorOnMissing

    • Type: boolean
    • Dynamic: ❓
    • Required: ❌
    • Default: false

    raise an error if the file is not found

    # host

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    Hostname of the remote server

    # insecureTrustAllCertificates

    • Type: boolean
    • Dynamic: ❓
    • Required: ❌

    Whether the client should disable checking of the remote SSL certificate.

    Note: This makes the SSL connection insecure, and should only be used for testing.

    # mode

    • Type: string

    • Dynamic: ❓

    • Required: ❌

    • Default: EXPLICIT

    • Possible Values:

      • IMPLICIT
      • EXPLICIT

    Sets FTPS mode, either "implicit" or "explicit".

    # passiveMode

    • Type: boolean
    • Dynamic: ❓
    • Required: ❌
    • Default: true

    Enter into passive mode.

    # password

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    Password on the remote server

    # port

    • Type: string
    • Dynamic: ❓
    • Required: ❌
    • Default: 990

    Port of the remote server

    # proxyHost

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    FTP proxy host

    # proxyPort

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    FTP proxy port

    # proxyType

    • Type: string

    • Dynamic: ❓

    • Required: ❌

    • Possible Values:

      • DIRECT
      • HTTP
      • SOCKS

    FTP proxy type

    # remoteIpVerification

    • Type: boolean
    • Dynamic: ❓
    • Required: ❌
    • Default: true

    Control that the server ip that emit the request is the same than send response.

    # rootDir

    • Type: boolean
    • Dynamic: ❓
    • Required: ❌
    • Default: true

    Is path is relative to root dir

    # uri

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    The file to delete

    # username

    • Type: string
    • Dynamic: ❓
    • Required: ❌

    Username on the remote server

    # Outputs

    # deleted

    • Type: boolean
    • Default: false

    If the files was really deleted

    # uri

    • Type: string

    The deleted uri
