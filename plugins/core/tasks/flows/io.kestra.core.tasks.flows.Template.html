Template 

type: "io.kestra.core.tasks.flows.Template"

Include a resuable template inside a flow

# Examples

id: template
namespace: io.kestra.tests

inputs:
  - name: with-string
    type: STRING

tasks:
  - id: 1-return
    type: io.kestra.core.tasks.debugs.Return
    format: "{{task.id}} > {{taskrun.startDate}}"
  - id: 2-template
    type: io.kestra.core.tasks.flows.Template
    namespace: io.kestra.tests
    templateId: template
    args:
      my-forward: "{{ inputs.with-string }}"
  - id: 3-end
    type: io.kestra.core.tasks.debugs.Return
    format: "{{task.id}} > {{taskrun.startDate}}"

# Inputs

# args

  • Type: object
  • SubType: string
  • Dynamic: ✔️
  • Required: ❌

The args to pass to the template

You can provide a list of named arguments (like function argument on dev) allowing to rename outputs of current flow for this template.
for example, if you declare this use of template like this:

  - id: 2-template
    type: io.kestra.core.tasks.flows.Template
    namespace: io.kestra.tests
    templateId: template
    args:
      forward: "{{ output.task-id.uri }}"

You will be able to get this output on the template with {{ parent.outputs.args.forward }}
